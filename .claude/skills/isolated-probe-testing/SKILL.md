# 隔离探测测试（Isolated Probe Testing）

> 不要在复杂系统里反复试错，要在最小环境中快速验证。

## 核心思想

复杂系统的问题往往由多个变量叠加：

- 代码逻辑
- 依赖行为
- 输入数据
- 系统状态

目标：一次只验证一个假设。

---

## Probe 定义

Probe 是最小实验单元：

必须包含：
- 输入
- 行为
- 输出
- 假设

要求：
- 独立运行
- 快速执行
- 可重复
- 单一验证目标

---

## 使用时机

- 多次修改仍失败
- 调试成本过高
- 不确定依赖行为
- 跨模块问题难定位

---

## 标准流程（SOP）

1. 拆解单一问题
2. 构造最小环境
3. 执行 Probe
4. 得出明确结论
5. 再集成回系统
6. 做端到端验证

---

## Probe 执行方式

按语言选择最小执行单元：

- Python：最小脚本 / -c
- Node.js：node file.js
- Go：go run main.go
- Java：main 方法
- Shell：直接命令

---

## 常见错误

- 一个 Probe 验证多个问题
- 依赖完整系统
- 输出不确定
- 只验证正常路径
