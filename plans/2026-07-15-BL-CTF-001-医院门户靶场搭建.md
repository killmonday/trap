# 2026-07-15 BL-CTF-001 医院门户靶场搭建

## 做什么

创建一个基于 Golang 的医院门户网站 CTF 靶场，参赛者通过目录扫描发现源码备份压缩包获取 flag。

## 为什么

CTF 比赛需要一道 Web 目录扫描类题目，模拟真实场景中的信息泄露漏洞。

## 怎么做

1. 使用 Go net/http + html/template + embed 构建无外部依赖的单体 Web 应用
2. 参考医院门户网站设计，制作 5 个页面（首页、科室、医生、简介、联系）
3. 在多个位置埋 hint：HTML hidden 元素、JS 注释/变量/console.log、robots.txt
4. 实现 /backup/ 路径的文件下载功能，含路径遍历防护
5. 编写 7 个测试用例覆盖页面访问、hint 验证、安全防护

## 关键设计决策

### Hint 策略：多线索交叉验证

- HTML 源码 hint（view-source 可发现）
- JS 注释和 console.log（F12 可发现）
- robots.txt（目录扫描工具自动检测）

三条线索指向同一路径 /backup/wwwroot.zip，确保做题人不会因错过某条线索而卡关。

### 干扰路径

robots.txt 中同时 Disallow /admin/、/config/、/api/ 三条假路径，增加扫描的干扰项。

### 安全防护

使用 filepath.Base 做文件名校验，防止路径遍历攻击读取系统文件。

## 验证结果

- 7/7 测试通过
- go build 成功
- 所有页面 HTTP 200
- Hint 在 HTML/JS/robots.txt 中均可被检索到
- 路径遍历攻击返回 404

## 风险

- 低：当前无后端数据库，表单提交仅前端校验无后端处理（符合题目设计意图）
